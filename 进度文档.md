# Eleven Memory Agent Platform（EMAP）进度文档

更新时间：2026-05-26

## 1. 当前进度总览

### 已完成

| 功能 | 实现方式 | 当前限制 |
|---|---|---|
| 健康检查 | `GET /health` 返回服务状态 | 仅基础存活检查，无下游依赖深度探测 |
| 一键启动脚本 | `scripts/start.ps1`（依赖同步 + 基础设施检查 + memory 迁移 + 服务启动） | 默认依赖本机可用的 MySQL/Redis 与 `uv` |
| 文档导入 | `POST /v1/ingest`，支持 `content` 或 `file_path` | 超大文档导入仍是同步处理 |
| 知识记忆删除 | `DELETE /v1/ingest/{document_id}`，执行 metadata + 向量索引清理 | 当前为单文档同步删除，未接异步物理清理队列 |
| 文件解析（`.md/.txt/.pdf`） | `LangChain + UnstructuredLoader` 本地解析 | 扫描版 PDF/OCR 未接入 |
| 文本分块 | `RecursiveCharacterTextSplitter` | 仍是通用分块策略，未做领域自适应 |
| 向量索引 | `FAISS(IndexIDMap2 + IndexFlatIP)` 本地持久化 | 当前为单机本地索引 |
| 真实 Embedding | `BAAI/bge-m3`（`langchain-huggingface`） | 首次下载体积大、预热耗时长 |
| 元数据存储 | MySQL `documents/chunks` 表 | 依赖外部 MySQL 可用 |
| MySQL 访问层 | SQLAlchemy 连接池 + `pymysql` 驱动 | 仍依赖外部 MySQL 可用 |
| 环境配置加载 | 启动时自动读取根目录 `.env` 并注入进程环境变量 | 仅支持 `KEY=VALUE` 简单格式，不解析复杂多行场景 |
| 检索接口 | `POST /v1/retrieve`，FAISS 召回 + hybrid 重排 | 重排仍为规则型，未接学习型 reranker |
| 引用返回 | 返回 `chunk_id/document_id/content/score` | 引用粒度是 chunk，非句级证据 |
| LLM 生成接入（可开关） | `qa.ask` 支持“证据注入 Prompt -> OpenAI 兼容 Chat Completions” | 需正确配置外部模型网关（如 MiMo Pro） |
| 索引重建 | `scripts/rebuild_faiss_index.py` | 需手动执行 |
| RAG 离线评估 | `scripts/evaluate_rag.py` + `evaluation/*`（默认本地指标，支持可选 RAGAS/Phoenix） | RAGAS 需额外模型配置；Phoenix 当前为文件导出模式 |
| 评估约束门禁 | 评估脚本支持 `min-*` 阈值约束，不达标返回非 0 | 当前仅覆盖通用质量指标，尚未覆盖业务自定义指标 |

### 部分完成

| 功能 | 实现方式 | 当前限制 |
|---|---|---|
| 问答能力 | `POST /v1/chat` 支持 LLM 生成与模板兜底双路径 | 真实效果依赖外部模型质量与网关稳定性 |
| 偏好记忆治理 | `POST/GET/DELETE /v1/memory/preferences` + `POST /v1/memory/preferences/rollback` + `GET /v1/memory/preferences/{user_id}/{key}/history`，MySQL 持久化 + 版本审计 | 当前仅覆盖偏好记忆，知识记忆治理仍未落地 |
| 会话记忆 | session 消息写入 Redis | 依赖外部 Redis 可用 |
| Memory 监控告警 | 操作耗时/失败计数指标 + 阈值告警（日志） | 当前为进程内指标，未接外部监控系统 |
| Memory 初始化脚本管理 | `scripts/manage_memory_schema.py` + `schema_migrations` 记录 | 当前仅覆盖 memory 模块脚本；v2 字段迁移已按列存在性做幂等处理 |
| Embedding 预热 | 启动时 warmup + `scripts/warmup_embedding.py` | 预热失败仅告警，不阻塞服务 |
| Metadata 重排加权 | 利用标题/层级/文件名等 metadata 进行规则加分 | 权重未做离线评估校准 |
| 评测集构建 | 支持 `.json/.jsonl` 数据集解析与字段兼容 | 仍需补充业务域黄金集规模与覆盖度 |

### 未完成

| 功能 | 实现方式 | 当前限制 |
|---|---|---|
| OCR 解析链路 | 暂无 | 扫描件不可直接解析 |
| MySQL 元数据层 | MySQL `documents/chunks` 表 | 生产并发与治理能力不足 |
| Redis 状态层 | 暂无（当前 Redis 独立连接） | 无跨实例会话治理与监控 |
| 结构化/语义化分块策略 | 暂无（当前 `RecursiveCharacterTextSplitter` 通用切分） | 尚未按标题/段落/表格/代码块等结构做差异化切分，复杂文档检索与引用精度受限 |
| 生产级检索栈 | 暂无 BM25/专业 reranker 服务 | 召回与排序稳定性有限 |
| 真实 LLM 回答生成 | 已支持 OpenAI 兼容协议接入（MiMo Pro 可配置） | 暂未覆盖流式输出、工具调用与细粒度输出约束 |

## 2. 已落地关键改造

1. 文档加载器改造：统一为 `LangChain + UnstructuredLoader`。
2. 分块链路改造：由手写滑窗切换为 `RecursiveCharacterTextSplitter`。
3. 向量层改造：由本地 JSON 哈希向量切换为 `FAISS + 真实 Embedding`。
4. 检索链路改造：改为 “FAISS 召回 + 规则重排（关键词 + token + metadata）”。
5. 数据库访问层改造：MySQL 切换为 SQLAlchemy 连接池，`memory` 与 `metadata` 共享同一池化客户端。
6. 工具链补齐：增加 embedding 预热脚本与 FAISS 索引重建脚本。
7. 目录分层映射：新增 `document_processing/embedding/vector_storage/qa` 适配层，保留现有服务实现不变。
8. 统一门面：新增 `agent_system.AgentSystem`，对外聚合解析、嵌入、向量检索与问答入口。
9. API 收口：控制器改为依赖 `AgentSystem`，减少对具体 service 的直接耦合。
10. 统一门面懒加载：避免模块导入时触发 MySQL 初始化，降低启动侧回归风险。
11. 逻辑迁移完成：`document_processing` 与 `qa` 承载核心流程，`services/*` 退化为兼容适配层。
12. 持久化升级补齐：`memory` 增加重试、连接池可观测、health 快照、告警阈值与初始化脚本管理。
13. 评估能力补齐：新增 `evaluation` 模块与 `scripts/evaluate_rag.py`，支持检索命中率、上下文精召回、引用覆盖率评估，并可选接入 `RAGAS` 与 `Phoenix` 导出。
14. 评估约束补齐：新增 `evaluation/constraints.py`，将评估测试固化为可执行阈值约束，支持 CI 门禁（失败返回码 2）。
15. 配置体验修复：`core/config.py` 增加 `.env` 自动加载逻辑，避免未显式导出环境变量时回落默认占位 DSN。
16. 启动体验补齐：新增 `scripts/start.ps1` 一键启动脚本，串联依赖同步、连通性检查、schema 应用与 API 启动。
17. 生成链路补齐：`qa.ask` 新增 Prompt 注入与 LLM 调用（可开关），失败自动回退模板答案，保持引用可追溯。
18. 偏好记忆治理补齐：新增偏好软删除与历史审计表（`user_preference_versions`），并开放 `delete/history` API。
19. 偏好回滚能力补齐：新增 `rollback` API 与仓储层回滚逻辑，可按版本或最近变更执行偏好值恢复并追加审计记录。
20. 知识记忆删除闭环补齐：新增文档级删除 API，删除时联动 `chunks` 元数据与 FAISS 索引映射清理，并在检索阶段过滤幽灵向量命中。

## 3. 当前主要风险

1. `BAAI/bge-m3` 首次下载依赖网络，可能导致预热时间不可控。
2. 记忆层已迁移为 MySQL + Redis，但依赖外部基础设施稳定可用。
3. OCR 和生产级检索组件未接入，复杂场景效果上限受限。
4. 评估集当前仅提供示例样本，尚未形成稳定回归基线集。
5. 约束阈值仍需结合真实业务数据校准，避免误报或漏报。
6. 一键启动脚本当前仅覆盖本机单实例启动，尚未覆盖容器/远程部署场景。
7. LLM 调用当前为同步阻塞请求，暂无流式返回和超时分级重试策略。

## 4. 下一阶段建议

1. Memory-first 落地：按 `Phase A/B/C/D` 先补齐 schema/API、四层运行时、长期治理与评测门禁，作为当前最高优先级。
1. 持久化升级：`memory` 继续补齐连接池、监控告警与初始化脚本管理。
2. 检索增强：向量索引与重排并行优化；索引侧补参数评测与分层检索策略，排序侧引入学习型 reranker，并将离线评测纳入发布前门禁。
3. 解析增强：补 OCR 流程，覆盖扫描件 PDF。
4. 生成增强：接入真实 LLM，完善回答质量与引用一致性校验。
5. 评估深化：补充 RAGAS 运行时模型配置与 Phoenix 在线 trace 回灌链路。
6. 约束深化：增加业务域指标约束（来源完整性、引用覆盖到句级、关键问答集通过率）。
7. 工程化增强：为 `scripts/start.ps1` 补充 CI smoke check 与跨平台启动脚本（bash）。
8. 生成体验增强：补流式输出、重试退避与输出格式约束（句级引用、拒答模板）。
