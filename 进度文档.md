## 2026-06-06 检索升级为真实混合召回

状态：已完成

本次变更：将检索主链从“FAISS + 简化 lexical/token 打分”升级为真正的 `BM25 + FAISS + CrossEncoder reranker` 混合召回/重排方案，并把召回路数、候选池大小、BM25 参数和融合权重全部配置化。

实现方式变化：新增 `qa/retrieval_stack.py`，用 Okapi BM25 公式做关键词召回，用 FAISS 做向量召回，用 `sentence-transformers` 的 `CrossEncoder` 做真实 reranker，再按 `HYBRID_BM25_WEIGHT/HYBRID_VECTOR_WEIGHT/HYBRID_RERANKER_WEIGHT` 融合最终分数；同时补充了检索回归测试。

当前限制：
- BM25 当前仍是基于 MySQL `chunks` 全量拉取后的进程内计算，召回质量是真 BM25，但规模能力还不是生产级全文索引。
- reranker 依赖 `BAAI/bge-reranker-v2-m3` 模型可下载或已缓存；首次运行若本地没有模型，会有预热/下载成本。

下一步建议：
1. 将 BM25 从进程内实现替换为独立全文检索能力（如 ES / OpenSearch / Tantivy 系方案），避免文档规模上来后每次全量扫描。
2. 给 reranker 增加预热脚本、健康检查和超时降级策略，避免模型首次加载拖慢首个请求。

## 2026-06-06 Safety / Alignment 第一阶段工程化

状态：已完成

本次变更：按 `input_guard -> output_guard -> risk_level/降级 -> 权限过滤 -> 安全评测集 -> 审计日志` 顺序，把安全与可控能力接入了问答主链。

实现方式变化：
- 新增 `guards/input_guard.py`，对提示词注入、敏感信息外泄、危险请求做输入拦截和风险分级。
- 新增 `guards/output_guard.py`，对无引用、伪造引用、敏感信息泄露做输出校验，不通过时自动降级成“证据式回答”。
- 在 `qa/answering.py` 中加入 `risk_level` 驱动的降级策略：高风险问题默认不走自由生成，只返回可引用证据摘要。
- 新增 `authz/access_control.py`，基于 `USER_DOC_PERMISSIONS` 做用户文档前缀权限过滤，并在 `/v1/chat` 支持传入 `doc_id_prefixes` 进一步收窄作用域。
- 新增 `audit/audit_logger.py`，将风险等级、权限决策、检索结果、输出降级等信息落为本地 JSONL 审计日志。
- 扩展 `evaluation` 数据集和指标，新增 `expected_refusal / required_citations / forbidden_chunk_ids / allowed_doc_prefixes`，并统计 `refusal_match_rate / citation_validity_rate / forbidden_chunk_leak_rate`。

当前限制：
- 输入审查当前仍以规则匹配为主，适合拦截明显风险请求，但对隐晦攻击和复杂越权语义还不够强。
- 权限过滤目前是“用户 -> 文档前缀”级别，不是完整的租户、角色、资源三级授权模型。
- 审计日志当前写本地 JSONL，适合开发和演示，不适合高并发生产观测。

下一步建议：
1. 将规则式 input guard 升级为“规则 + 分类模型”双层检测，降低绕过率和误杀率。
2. 将权限模型升级到 chunk 级 metadata ACL，并补充管理端配置入口。
3. 将审计日志接到集中式观测链路，并增加风险告警和追踪检索。
