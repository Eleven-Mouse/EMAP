# 奶龙双模式 RAG 项目需求文档（MVP+）

## 1. 目标与约束

### 目标

- 构建一个“奶龙长期对话 + 学习型 RAG”一体化系统。
- 在同一会话中支持两种模式：
  - 陪聊模式：以奶龙人格风格对话，注重情绪价值与陪伴感。
  - 学习模式：基于知识库检索与引用回答，注重准确性与可追溯。
- 系统具备长期记忆能力（可管理、可删除、可追溯），支持持续学习与个性化。

### 约束

- 优先最小可用版本（MVP），先打通主链路再扩展能力。
- 记忆分层固定为三类：偏好记忆、知识记忆、会话记忆。
- 检索必须使用混合策略，不允许仅依赖向量召回。
- 学习模式必须返回来源引用，禁止无依据强答。
- 记忆能力必须支持开关、删除、编辑和延迟生效说明。

### 默认方案

- 后端框架：`Python + FastAPI + LangChain`
- 结构化数据：`MVP 本地 SQLite -> 生产 MySQL`
- 向量检索：`MVP 本地持久索引 -> 生产独立向量库Faiss`
- 短时状态与缓存：`Redis`
- 异步链路：`MQ`（后续可选，用于解析/OCR/批处理）

## 2. 产品形态与核心能力

### 2.1 双模式对话

- 陪聊模式（`social_chat`）：
  - 目标：人格化自然对话、轻松陪伴。
  - 行为：默认不强依赖知识库检索，可参考用户偏好记忆。
- 学习模式（`learn_qa`）：
  - 目标：高质量学习辅助。
  - 行为：强制走 RAG 检索链路，输出带引用答案。

### 2.2 意图路由

- 用户输入先进入意图路由器：
  - `social_chat`（闲聊）
  - `learn_qa`（学习问答）
  - `memory_cmd`（记忆管理指令）
- 路由策略：显式命令优先（如 `/聊`、`/学`），分类器兜底。

### 2.3 记忆分层

- 偏好记忆：语气风格、学习偏好、表达长度等。
- 知识记忆：导入文档及其结构化索引。
- 会话记忆：短时上下文与会话摘要。

## 3. 核心设计原则

- 记忆分层明确，职责不混淆。
- 学习模式回答必须保留来源引用，确保可审计、可追溯。
- 数据必须可编辑、可删除、可追溯；删除需保证主存/索引/缓存一致性。
- 禁止将全部历史会话直接塞入上下文，优先检索、摘要和短时状态管理。
- 对宽泛问题提供兜底策略，但需标注证据强弱。

## 4. 系统能力分解（双模式 RAG）

1. 数据源：文档、网页、数据库、FAQ、个人笔记。
2. 文档解析：支持 `.md/.txt/.pdf`，后续扩展 OCR。
3. 文档切分：结构优先 + 字符窗口兜底。
4. 向量化：embedding 或稳定哈希向量（MVP）。
5. 索引存储：元数据与向量分离存储。
6. 检索层：关键词召回 + 向量召回 + 重排（rerank）。
7. 意图路由：学习/闲聊/记忆命令分流。
8. 生成层：
   - 学习模式：证据优先 + 引用输出。
   - 陪聊模式：人格化表达优先。
9. 记忆与状态：短时会话状态 + 长期偏好记忆。
10. 评估监控：召回率、引用完整性、路由准确率、延迟与成本。

## 5. 存储与数据组织

- 偏好记忆：结构化存储，如语言偏好、输出风格、长度偏好。
- 知识记忆：文档 -> chunk -> 概念 -> 关系。
- 会话记忆：短时状态缓存，按策略过期或压缩。
- 反馈数据：记录有用/无用、纠错点，用于迭代检索与生成质量。

## 6. 学习模式核心流程

1. 解析新内容并完成切分入库。
2. 混合检索（关键词 + 向量 + 重排）。
3. 证据筛选与上下文构建。
4. 生成学习答案并输出引用。
5. 根据用户反馈更新记忆与检索策略。

## 7. 陪聊模式核心流程

1. 识别为 `social_chat` 意图。
2. 拉取用户偏好与最近会话摘要。
3. 奶龙人格化回复（不强制走知识检索）。
4. 更新会话记忆并保留必要偏好信号。

## 8. MVP 分阶段实施顺序

1. 阶段 1：导入、持久化、混合检索、引用问答主链路。
2. 阶段 2：意图路由（/学、/聊、自动分类兜底）。
3. 阶段 3：偏好记忆与会话记忆治理（开关/删改查）。
4. 阶段 4：学习模板增强（总结、练习、复习建议）。
5. 阶段 5：多渠道接入（CLI -> QQ）。

## 9. 最小验收清单

- 导入后可稳定召回目标片段。
- 学习模式回答包含有效来源引用。
- 陪聊模式体现奶龙人格风格，且能连续多轮。
- 同一会话可从陪聊切换到学习，再切回陪聊。
- 删除知识后不可继续被召回。
- 偏好记忆可影响回答风格。
- 记忆支持开关、删除与追溯。

## 10. 风险与治理

- 路由误判风险：学习问题被当闲聊。
  - 方案：显式命令优先 + 人工纠错指令（如“切到学习模式”）。
- 陪聊幻觉风险：闲聊中出现事实性错误。
  - 方案：涉及事实性问题自动升为学习模式。
- 记忆污染风险：错误偏好被长期固化。
  - 方案：记忆可回看、可删除、可禁用。

## 11. 版本目标（MVP+）

- 打造“可陪伴、可学习、可成长”的奶龙智能体：
  - 闲聊时像朋友，学习时像助教；
  - 回答可追溯，记忆可管理；
  - 支持长期演进到生产级 RAG 架构。
